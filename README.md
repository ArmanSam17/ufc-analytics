# UFC Fight Analytics & Outcome Prediction

An end-to-end data science project that ingests raw UFC fight records into a
relational database, explores 6,000+ bouts with advanced SQL and visualisations,
and trains machine learning models to predict fight outcomes from pre-fight
differential statistics. Built as a portfolio project demonstrating the full
analytics stack — from raw CSV to a calibrated ML model.

|---|---|---|

| `events` | 555 | Unique UFC events — date and location |
| `bouts` | 6,014 | One row per fight — result, method, referee, weight class, title flag |

## Key Findings

**1. Heavyweight finishes 70% of fights — Women's Strawweight only 31%**
Finish rate scales almost perfectly with weight class. Heavyweight leads at 69.8% (KO/TKO + submission), while Women's Strawweight goes to the judges 68.7% of the time, making it the most technical division in the promotion.

**2. Reach advantage is real but modest (r = 0.067)**
Fighters with a longer reach win at a slightly higher rate, but the Pearson correlation is only 0.067. The bucketed analysis shows the biggest edge from a +5–+10 cm advantage; beyond 10 cm the benefit plateaus. Reach is a useful scouting variable, not a deterministic predictor.

**3. UFC fights got longer through the 2010s, then stabilised**
Average fight duration climbed steadily from ~3 minutes in the early UFC era to ~10–11 minutes by 2015, as athletes became well-rounded and grappling counters to striking evolved. The mean has stabilised since, suggesting an equilibrium between offensive and defensive skill development.

**4. Southpaw fighters win 51.8% of bouts vs 48.6% for Orthodox**
A consistent ~3 percentage point edge — confirmed across 2,398 Southpaw appearances with tight bootstrap confidence intervals. The structural cause: every Southpaw trains against Orthodox opponents, but most Orthodox fighters rarely face a Southpaw in camp.

**5. Winners land 70% more significant strikes than losers (median 34 vs 20)**
The violin plots show clear distributional separation between winning and losing performance. `slpm_diff` (strikes landed per minute differential) is the single strongest predictor in the ML model, with a Pearson correlation of +0.25 with the Red-win target.

## Machine Learning Results

**Target:** 1 = Red corner wins, 0 = Blue corner wins  
**Features:** 9 differential stats (Red − Blue) — reach, height, age, win-pct, finish-rate, slpm, striking accuracy, takedown average, days rest  
**Class balance:** Red wins 67.4% of bouts (naive baseline accuracy)  
**Validation:** Stratified 5-Fold Cross-Validation

| Model | ROC-AUC | Accuracy | vs. Naive Baseline |
|---|---|---|---|
| Logistic Regression | 0.697 ± 0.017 | 70.4% | +3.0 pp |
| **Random Forest** ✓ | **0.727 ± 0.015** | **70.7%** | **+3.3 pp** |
| Gradient Boosting | 0.724 ± 0.016 | 70.1% | +2.7 pp |

The Random Forest was selected as the best model. Top features by importance: `slpm_diff`, `str_acc_diff`, `age_diff`, `td_avg_diff`, `win_pct_diff`. Physical differentials (reach, height) provided weak signal individually, consistent with the EDA correlation analysis.

> **Note on the accuracy ceiling:** Professional consensus places the upper bound for UFC fight prediction at ~65–70% accuracy. Key factors like gameplan, injury status, weight-cut severity, and camp quality are not captured in any public dataset, creating irreducible noise.

## SQL Skills Demonstrated

`02_sql_queries.py` runs 6 queries against `ufc.db`, each targeting a different SQL concept:

| # | Concept | Query |
|---|---|---|
| 1 | **Window function — RANK() OVER PARTITION** | Rank top-10 finishers per weight class by KO/TKO + submission rate (min 5 wins) |
| 2 | **CTE — multi-step aggregation** | Compute each fighter's current win streak using cumulative sums and a `shift(1)` chain |
| 3 | **Aggregation + multi-table JOIN** | Per weight class: avg fight duration (parsed from `M:SS`), avg sig. strikes, finish rate |
| 4 | **Window function — LAG()** | Days of rest between consecutive fights per fighter; returns busiest 25 athletes |
| 5 | **Multi-table JOIN (self-join pattern)** | Title fight "tale of the tape" — joins `fighters` twice (Red and Blue) onto every `title_bout = 1` row |
| 6 | **Chained CTEs — classification logic** | CTE 1 pulls career metrics; CTE 2 classifies fighters into styles (Grappler, Pressure Fighter, Counter Striker, Balanced); final SELECT aggregates finish rate per style |

## How to Run

**Requirements:** Python 3.10+, Git

```bash
# 1. Clone and enter the project
git clone <repo-url>
cd ufc-analytics

# 2. Create a virtual environment and install dependencies
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install pandas matplotlib seaborn scikit-learn scipy jupyter nbformat nbconvert

# 3. Build the SQLite database from raw CSVs
python 01_load_data.py
# → Creates ufc.db with 4 tables (fighters, events, bouts, bout_stats)
# → Prints row count verification for each table

# 4. Run the SQL analysis
python 02_sql_queries.py
# → Executes 6 queries and prints formatted results to stdout

# 5. Open the EDA notebook
jupyter notebook 03_eda.ipynb
# → 7 sections: physical attributes, reach advantage, finish rates,
#   fight duration trends, stance analysis, striking stats, key findings

# 6. Open the ML notebook
jupyter notebook 04_model.ipynb
# → Feature engineering, 3 models with 5-fold CV, Random Forest deep dive
#   (ROC curve, confusion matrix, calibration plot, feature importance)
```

All notebooks ship with pre-executed outputs — you can read them without running any code.

## Dataset

**Source:** [UFC Fight Historical Data from 1993 to 2021](https://www.kaggle.com/datasets/rajeevw/ufcdata) by **Rajeev Warrier** on Kaggle.

Raw files: `raw_fighter_details.csv` (fighter profiles) and `raw_total_fight_data.csv` (per-fight statistics). The `data.csv` and `preprocessed_data.csv` files are derived datasets also provided by the same author with pre-computed rolling averages and one-hot encodings.
